# Grain Datastar UI

`ai.obney.grain.datastar.ui` is an IR-first UI layer for Grain's Datastar
integration. Apps can alias it as `ui` when there is no local `ui` namespace:

```clojure
(:require [ai.obney.grain.datastar.ui :as ui])
```

## Pipeline

Datastar UI does not render HTML. It compiles checked UI source to IR, lowers
that IR to ordinary Datastar hiccup, and leaves HTML serialization to the
existing Datastar adapter.

```text
Datastar UI source
  -> UI IR
  -> plain Datastar hiccup
  -> Grain Datastar adapter
  -> hiccup2 HTML rendering / SSE patch
```

Use `ui/hiccup` for normal app code:

```clojure
(ui/hiccup
  [:button {:on/click
            (ui/dispatch :todo/complete-task {:task-id task-id})}
   "Complete"])
```

The result is plain hiccup with Datastar attrs, suitable for `:datastar/hiccup`.

## Explicit Payload Rule

Server effects generated by this namespace use explicit payloads. Ambient
Datastar signals are client-local UI state unless explicitly referenced in a
`dispatch` or `refresh` payload.

```clojure
(ui/dispatch :academic/create-campus
  {:campus-name campus-name}
  {:post "/actions"})

(ui/refresh "/admin/graduation-pending/__stream"
  {:page page
   :search search})
```

These lower to `@post(..., {payload: ...})`; they do not post all page signals.
`refresh` includes `dsNonce` by default as explicit reusable-stream protocol
metadata so the POST updates the existing SSE connection. Pass
`{:include-nonce? false}` for one-shot/manual interop. The low-level Datastar
adapter still accepts flat or legacy wrapped signal bodies for interop.

## Signals

Signals are lexical handles, not strings. `with-signals` attaches declarations
to the returned subtree root, and `with-signal-scope` makes generated names
deterministic and collision-resistant.

```clojure
(ui/with-signal-scope {:prefix "plan" :key application-id}
  (ui/with-signals [duration (ui/signal {:name "duration-weeks" :init 12})]
    [:input {:bind/value duration}]))
```

Signal references choose the correct Datastar syntax automatically. Identifier
names lower to `$name`; dashed or otherwise unsafe names lower to bracket
notation such as `$[\"plan-duration-weeks__abc\"]`.

## Checked Vocabulary

Datastar UI interprets only its own vocabulary:

- `:bind/value`, `:bind/text`, `:bind/show`, `:bind/class`, `:bind/attr`
- `:on/click`, `:on/change`, `:on/input`, `:on/submit`, `:on/keydown`
- effects such as `dispatch`, `refresh`, `set-signal!`, `reset!`, `when!`,
  `if!`, `do!`, `on-keys`, and `action`
- expressions such as `sig`, `trimmed`, `num`, `num-cents`, `present?`,
  `changed?`, `evt`, `lit`, and `js`

`dispatch` validates literal command names and map keys against Malli command
schemas when the schema is a registered `[:map ...]` shape. Runtime values
remain runtime values; raw JavaScript is intentionally opaque.

## Raw Datastar Passthrough

All non-DSL attrs pass through unchanged, including advanced production attrs:

```clojure
:data-signals
"data-signals__ifmissing"
"data-on-signal-patch"
"data-effect"
"data-indicator"
:data-ignore-morph
"data-attr:disabled"
:data-attr-disabled
:data-init
:data-on:input__debounce.500ms
```

Use `ui/action` for raw Datastar actions that should remain unchecked:

```clojure
{:on/click (ui/action "@post('/custom/path')")}
```

## IR

`ui/ir` returns Datastar-shaped IR. The IR is stable enough for tests and
tooling, but app code should usually consume `ui/hiccup`.

Example dispatch IR:

```clojure
{:op :dispatch
 :command :financial/record-arbitrary-refund
 :args {:amount-cents {:op :num-cents
                       :expr {:op :sig :name "amount-dollars"}}}
 :post "/actions"}
```

Example signal IR:

```clojure
{:op :signal
 :binding 'duration
 :semantic-name "duration-weeks"
 :resolved-name "plan-duration-weeks__abc"
 :scope {:prefix "plan" :key application-id}
 :init 12
 :type :int}
```

## Static Interpretation

`ui/static` interprets IR for static galleries. It drops checked events and
signal declarations, and can optionally remove `href` and raw `data-on*` attrs:

```clojure
(ui/static (ui/ir component)
  {:strip-href? true
   :strip-raw-events? true})
```

Raw JavaScript, custom modules, and raw actions remain opaque. Static and future
simulation coverage applies only to the checked vocabulary.
